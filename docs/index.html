Open source instructional design skills for Claude Code. Every recommendation is backed by peer-reviewed research, so you always know how strong the evidence is.

What it does

Your course, reviewed by research

idstack is a design partner that lives inside Claude Code. Import your course, and it checks your work against peer-reviewed research across 11 domains. No jargon. No guessing. Just evidence.

  • ✓ Quality review against Quality Matters standards and Community of Inquiry framework
  • ✓ Alignment check between your learning objectives, assessments, and activities
  • ✓ Cognitive load flags when modules overload working memory
  • ✓ Bloom's classification of every learning objective, automatically
  • ✓ Evidence strength for every recommendation, from T1 (meta-analyses) to T5 (expert opinion)
  • ✓ Accessibility audit with WCAG compliance and Universal Design for Learning review

Get started

Up and running in 5 minutes

$ git clone https://github.com/savvides/idstack.git && cd idstack && ./setup

New to Claude Code? No problem. Here's everything, step by step.

  1. Install Claude Code. It's a free AI tool that runs in your terminal.
    Visit claude.ai/download and follow the instructions for your computer.
  2. Install idstack. Open your terminal and paste the command above, or:
    git clone https://github.com/savvides/idstack.git && cd idstack && ./setup
  3. Start Claude Code by typing claude in your terminal.
  4. Import your course. Type /idstack:course-import and paste your syllabus, upload a Common Cartridge file, or connect your Canvas API.
  5. Get your review. Type /idstack:course-quality-review to see your evidence-based course audit. Or type /idstack:pipeline to run the full design pipeline automatically.
Your design team

11 specialists. One command.

Each skill is a specialist on your team. Run /idstack:pipeline and they work in sequence, or invoke any one directly.

Needs Analyst
/idstack:needs-analysis
Three-level assessment before you build anything. Is training the right intervention? Task analysis and learner profiling.
Curriculum Designer
/idstack:learning-objectives
Measurable objectives with Bloom's taxonomy. Bidirectional alignment check: does each objective have a matching activity AND assessment?
Assessment Architect
/idstack:assessment-design
Evidence-based rubrics and feedback strategies. Formative checkpoints before summative assessments. Nicol's 7 principles of good feedback.
LMS Bridge
/idstack:course-import
Import from Common Cartridge, Canvas API, SCORM, PDF, or paste. Quick-scan quality flags and Bloom's pre-classification on import.
Content Generator
/idstack:course-builder
Generates syllabus, module pages, assignments, and rubrics from your manifest. Content adapts to learner expertise level.
Quality Auditor
/idstack:course-quality-review
Full Quality Matters audit plus Community of Inquiry presence analysis. 8 structural standards, 3 presence dimensions. Every finding cites its evidence.
Accessibility Reviewer
/idstack:accessibility-review
WCAG 2.1 AA compliance audit plus Universal Design for Learning review. Scores accessibility 0-100 with "Must Fix" and "Should Improve" tiers.
Adversarial Auditor
/idstack:red-team
Assumes your course is broken and tries to prove it. Alignment stress test, persona simulation, prerequisite chain integrity. Confidence score.
LMS Publisher
/idstack:course-export
Exports to IMS Common Cartridge, SCORM 1.2, or pushes directly to Canvas via API. Shows readiness dashboard before export.
Pipeline Orchestrator
/idstack:pipeline
Chains all 8 skills automatically in order. Auto-skips completed skills. Pause anytime, resume later. One command for the full design cycle.
Memory Manager
/idstack:learn
Search, delete, promote, and export learnings. Cross-project intelligence means discoveries from one course help design the next.

How they fit together

Skills are plain Markdown files. No build step, no dependencies. Each skill reads the shared manifest and session history, runs its evidence-based workflow, and logs the result. Your course context builds up over time.

Where we're headed

Right now idstack covers the core loop: analyze needs, write objectives, design assessments, build content, and review quality. That handles most of what you do day to day. But there's more to get to.

The next skills will handle the specialized workflows that eat up the rest of your time: sequencing content so it doesn't overload learners, choosing the right media for each objective, and planning how to measure whether your course actually worked.

Longer term, the goal is pushing changes back to your LMS. You update an objective in idstack, and it updates in Canvas. Every connected assessment and module page updates with it. No more maintaining two copies of your course.
