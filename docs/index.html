What it does

Your course, reviewed by research

idstack is a design partner that lives inside Claude Code. Import your course, and it checks your work against peer-reviewed research across 11 domains. No jargon. No guessing. Just evidence.

  • ✓ Quality review against Quality Matters standards and Community of Inquiry framework
  • ✓ Alignment check between your learning objectives, assessments, and activities
  • ✓ Cognitive load flags when modules overload working memory
  • ✓ Bloom's classification of every learning objective, automatically
  • ✓ Evidence strength for every recommendation, from T1 (meta-analyses) to T5 (expert opinion)
  • ✓ Accessibility audit with WCAG compliance and Universal Design for Learning review

Get started

Up and running in 5 minutes

$ git clone https://github.com/savvides/idstack.git && cd idstack && ./setup

New to Claude Code? No problem. Here's everything, step by step.

  1. Install Claude Code. It's a free AI tool that runs in your terminal.
    Visit claude.ai/download and follow the instructions for your computer.
  2. Install idstack. Open your terminal and paste the command above, or:
    git clone https://github.com/savvides/idstack.git && cd idstack && ./setup
  3. Start Claude Code by typing claude in your terminal.
  4. Import your course. Type /course-import and paste your syllabus, upload a Common Cartridge file, or connect your Canvas API.
  5. Get your review. Type /course-quality-review to see your evidence-based course audit. You'll know in 5 minutes if this is for you.
How it works

9 skills. One manifest. Zero dependencies.

Skills are plain Markdown files that form a pipeline. Each reads from and writes to a shared project manifest. No build step. No runtime dependencies.

The Pipeline

/needs-analysis → /learning-objectives → /assessment-design
↓
/course-builder → /course-quality-review → /accessibility-review
↓
/red-team → /course-export

/course-import → enters at /learning-objectives

Architecture

  • ✓ Each skill is a directory with a single SKILL.md file (YAML frontmatter + Markdown workflow)
  • ✓ Shared state through .idstack/project.json manifest. Each skill owns its section, preserves the rest.
  • ✓ Every recommendation cites its evidence tier using domain codes from evidence/references.md
  • ✓ Works standalone or as a pipeline. Skills ask questions directly if no manifest exists.
  • ✓ Import from Common Cartridge, Canvas API, SCORM, PDF, or plain text. Export to IMS CC or SCORM.
